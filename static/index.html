JSONPlaceholder Clone

A RESTful API service built with Go - Perfect for testing and prototyping

Welcome

This is a Go implementation of JSONPlaceholder, providing a fake REST API for testing and prototyping. All data is hard-coded and static. Write operations (POST, PUT, PATCH, DELETE) return mocked responses but do not persist data.

Note: This API is for development and testing purposes. No data is actually stored or modified.

Query Parameters

Filtering

Filter results by any field using query parameters:

GET /posts?userId=1
GET /comments?postId=1
GET /albums?userId=2
GET /photos?albumId=1
GET /todos?userId=1&completed=true

Pagination

Use _page and _limit to paginate results:

GET /posts?_page=1&_limit=2
GET /comments?_page=2&_limit=3
  • _page: Page number (default: 1)
  • _limit: Items per page (default: all items)

Artificial Delay

Add artificial delay to responses (useful for testing loading states):

GET /posts?_delay=2000
  • _delay: Delay in milliseconds (max: 10000ms)

Example cURL Commands

Get all posts

curl http://localhost:3000/posts

Get a specific post

curl http://localhost:3000/posts/1

Get comments for a post

curl http://localhost:3000/posts/1/comments

Filter posts by user

curl http://localhost:3000/posts?userId=1

Paginate results

curl http://localhost:3000/posts?_page=1&_limit=2

Create a new post (mocked)

curl -X POST http://localhost:3000/posts \
  -H "Content-Type: application/json" \
  -d '{
    "userId": 1,
    "title": "My new post",
    "body": "This is the content of my post"
  }'

Update a post (PUT - mocked)

curl -X PUT http://localhost:3000/posts/1 \
  -H "Content-Type: application/json" \
  -d '{
    "userId": 1,
    "title": "Updated title",
    "body": "Updated body"
  }'

Patch a post (PATCH - mocked)

curl -X PATCH http://localhost:3000/posts/1 \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Partially updated title"
  }'

Delete a post (mocked)

curl -X DELETE http://localhost:3000/posts/1

Add artificial delay

curl http://localhost:3000/posts?_delay=2000

Response Format

All API responses use Content-Type: application/json with UTF-8 encoding.

Success Response

Successful requests return the requested resource(s) with appropriate HTTP status codes:

  • 200 OK: GET, PUT, PATCH, DELETE operations
  • 201 Created: POST operations

Error Response

Errors return a JSON object with an error message:

{
  "error": "not found"
}
  • 400 Bad Request: Invalid ID or malformed JSON
  • 404 Not Found: Resource does not exist

Important Notes

  • All data is hard-coded and static - only 5 items exist per resource type
  • Write operations (POST, PUT, PATCH, DELETE) return mocked responses but do not persist data
  • POST operations always return a new ID of 6 (since we have 5 hard-coded items)
  • No authentication or rate limiting is implemented
  • This API is intended for development and testing purposes only
